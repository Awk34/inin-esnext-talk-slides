Default, Rest, Spread 


function f(x, y=12, z=(() => 3)()) {
    // y is 12 if not passed (or passed as undefined)
    return x + y + z;
}
f(3) === 18
            

function f(x, ...y) {
    // y is an Array
    return x * y.length;
}
f(3, "hello", true) === 6
            

function f(x, y, z) {
    return x + y + z;
}
// Pass each elem of array as argument
f(...[1,2,3]) === 6

    yield 0;
    yield* countToThree();
    yield 4;

    yield index++;
}

    yield 0;
    yield* countToThree();

for(let n of zeroToFour()) {

Symbols

A symbol is a unique and immutable data type and may be used as an identifier for object properties.

There is a Symbol primitive data type, and a globally available Symbol implicit object wrapper for it.

Well-known Symbols:

  • Symbol.iterator
  • Symbol.hasInstance
  • Symbol.toPrimitive

Symbol('foo') !== Symbol('foo')
like named enumerations in C

The Set object lets you store unique values of any type, whether primitive values or object references.

WeakMaps   & WeakSets

Same as Maps and Sets, except they don't prevent their keys/elements from being garbage-collected (never worry about memory leaks!).

Don't allow iteration/looping (volatility) or clearing (security).

WeakSets are of Objects only
